# LLM Temperature 最佳实践

## 什么是 Temperature？

Temperature 是控制 LLM 输出随机性的参数（范围 0.0-2.0）：

- **低值**：更确定、一致、保守
- **高值**：更随机、创意、多样化

---

## 推荐设置

| 场景 | Temperature | 说明 |
|------|-------------|------|
| 代码生成、数据提取 | **0.0 - 0.3** | 确定性输出，可测试，避免幻觉 |
| 技术文档、企业内容 | **0.1 - 0.3** | 保持一致性，适合合规场景 |
| 一般写作、产品文案 | **0.4 - 0.7** | 平衡多样性与一致性 |
| 对话系统、聊天机器人 | **0.6 - 0.8** | 自然流畅，偶尔有变化 |
| 头脑风暴、创意写作 | **0.8 - 1.2** | 更多多样性，需人工审核 |
| 命名、Slogan 生成 | **0.8 - 1.0** | 鼓励新颖组合 |

---

## 关键原则

### 1. 生产环境默认值

```typescript
// 企业/生产推荐
const config = {
  temperature: 0.2,  // 保守但可用
  topP: 0.9,
};

// 创意场景
const creativeConfig = {
  temperature: 0.7,
  topP: 0.95,
};
```

### 2. 不要同时调整 temperature 和 top-p

- OpenAI 和 Anthropic 都建议：**只调其中一个**
- 推荐先调 temperature，必要时再调 top-p

### 3. 按模型调整

| 模型 | 推荐范围 |
|------|----------|
| GPT-4 / GPT-4o | 0.5 - 0.7（大多数场景） |
| Claude 系列 | 默认 1.0，生产建议 0.2 - 0.7 |
| o1 / o3（推理模型） | 0.1 - 0.3（已深度思考） |

### 4. 避免极端值

- **> 1.5**：输出可能变得混乱、不连贯
- **= 0.0**：完全确定性，适合测试但可能过于死板

---

## 实际示例对比

**Prompt**: "用量子计算打个比方"

| Temperature | 输出风格 |
|-------------|----------|
| 0.2 | "量子计算使用量子比特，可以同时处于 0 和 1 的状态..." |
| 0.7 | "想象一枚硬币可以同时是正面和反面..." |
| 1.2 | "就像让薛定谔的猫来抛掷概率..." |

---

## 企业级建议

1. **审计与合规**：使用 0.0 - 0.3，确保输出可预测
2. **A/B 测试**：用工具（如 promptfoo）系统测试不同 temperature
3. **监控质量**：低 temperature 不保证准确性，只是减少随机性
4. **固定种子**：如需完全可复现，结合 temperature=0 使用

---

## 常见误区

| 误区 | 真相 |
|------|------|
| 低 temperature = 更准确 | 只是更确定，不一定更准确 |
| 高 temperature = 更有创意 | 超过 1.0 可能产生噪声 |
| 同时调 temperature 和 top-p | 选择其中一个作为主控 |

---

## 快速参考

```typescript
// 代码/数据任务
const codingConfig = { temperature: 0.2 };

// 一般对话
const chatConfig = { temperature: 0.7 };

// 创意任务
const creativeConfig = { temperature: 0.9 };
```

---

> **总结**：对于大多数生产应用，**0.2 - 0.7** 是最佳区间。代码/数据任务用低端，创意任务用高端，始终根据实际效果测试调整。
